Plan 모드는 복잡한 작업 전 계획을 수립하고 사용자 승인을 받는 단계입니다.

## 진입 조건

- 3개 이상 파일 수정 예상
- 아키텍처 결정이 필요한 작업
- 여러 접근법이 가능한 경우
- 불명확한 요구사항

## 워크플로우

1. **EnterPlanMode** 도구로 진입
2. 코드베이스 탐색 및 분석
3. 계획 파일에 접근법 작성
4. **계획이 만족스러울 때까지 반복 협의**
5. ExitPlanMode로 구현 시작

## 핵심 원칙

- 계획 확정 전까지 코드 작성 금지
- 사용자 피드백 적극 반영
- 트레이드오프 명시
- 구현 순서 정의

## Plan 모드 vs auto-dev SPECIFY

| 항목 | Plan 모드 | auto-dev SPECIFY |
|------|----------|------------------|
| 목적 | 구현 방법 설계 | 요구사항 확정 |
| 결과물 | 구현 계획 | 완료 조건 |
| 적합한 경우 | 설계 불확실 | 요구사항 불확실 |
