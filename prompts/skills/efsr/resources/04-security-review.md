# 자체 보안성심의 (제36조)

> 근거: 전자금융감독규정 제36조
> 시행일: 2025.2.5
> 분류: **금융 고유 요건** (ISMS-P에 없음)

---

## 1. 개요

자체 보안성심의는 **금융권 고유의 사전 보안 검증 절차**입니다. 신규 전자금융업무 출시 전 보안성을 자체 심의하고, 결과를 금융감독원에 제출하여 검토받는 제도입니다.

ISMS-P에는 동등한 절차가 없으며, 금융회사 및 전자금융업자에게만 적용됩니다.

---

## 2. 심의 대상 (제36조 제1항)

### 2.1 필수 심의 대상

| 구분 | 대상 | 예시 |
|------|------|------|
| **신규 전자금융업무** | 정보통신망을 이용하여 이용자를 대상으로 수행하는 신규 전자금융업무 | 모바일뱅킹 신규 기능, 간편결제 서비스, 오픈뱅킹 연동 |
| **공동 표준 제정** | 복수의 금융회사 또는 전자금융업자가 공동으로 제정하는 전자금융거래 관련 표준 | 금융권 공통 인증 표준, 데이터 교환 프로토콜 |

### 2.2 심의 대상 판단 기준

```
신규 여부 판단:
├─ 신규 서비스 출시 → 심의 대상
├─ 기존 서비스 기능 추가 → 심의 대상 (이용자 대상, 정보통신망 이용)
├─ 단순 UI 변경 → 심의 대상 아님
├─ 내부 업무 시스템 → 심의 대상 아님 (이용자 대상 아님)
└─ 기존 기능 유지보수 → 심의 대상 아님
```

---

## 3. 심의 절차 (금감원장이 정하는 기준)

### 3.1 표준 절차

```
┌─────────────────────────────────────────────────────────────────┐
│  1. 심의 대상 식별                                               │
│     └─ 신규 전자금융업무 또는 공동 표준 해당 여부 판단            │
├─────────────────────────────────────────────────────────────────┤
│  2. 자체 보안성심의 실시                                         │
│     └─ 금감원장이 정하는 기준과 절차에 따라 심의                  │
├─────────────────────────────────────────────────────────────────┤
│  3. 결과보고서 작성                                              │
│     └─ 심의 결과, 보안 조치 사항 문서화                          │
├─────────────────────────────────────────────────────────────────┤
│  4. 금감원 제출 (심의 완료 후 7일 이내)                          │
│     └─ 결과보고서 제출 (면제 기관 제외)                          │
├─────────────────────────────────────────────────────────────────┤
│  5. 금감원 검토                                                  │
│     └─ 보안수준 충분성 검토                                      │
├─────────────────────────────────────────────────────────────────┤
│  6. 개선/보완 조치 (필요 시)                                     │
│     └─ 금감원 요구에 따른 보완 조치 이행                         │
├─────────────────────────────────────────────────────────────────┤
│  7. 서비스 출시                                                  │
│     └─ 보안성심의 완료 후 서비스 개시                            │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 심의 주체

- **정보보호최고책임자(CISO)**: 자체심의 총괄 (제8조의3 제5호)
- **정보보호위원회**: 중요 사항 심의 및 의결 (제8조의2)

---

## 4. 제출 요건 (제36조 제2항)

### 4.1 제출 기한

| 항목 | 내용 |
|------|------|
| 제출 시점 | 자체 보안성심의 완료 후 **7일 이내** |
| 제출 대상 | 금융감독원 |
| 제출 문서 | 결과보고서 |

### 4.2 제출 면제 조건

다음 조건을 **모두** 충족하는 경우 제출 면제:

1. 과거 **1년 이내 전자금융사고 미발생**
2. 금융감독원장이 정하는 기준에 해당

### 4.3 제출 면제 기관 (제36조 제4항)

다음 기관은 보고서 제출 의무 없음:

| 기관 | 근거 |
|------|------|
| 체신관서 | 우체국예금 및 보험에 관한 법률 |
| 새마을금고 및 새마을금고중앙회 | - |
| 한국수출입은행 | - |
| 공공기관 | 공공기관의 운영에 관한 법률 제4조 |

**주의**: 제출 면제 기관도 **자체 보안성심의 자체는 실시**해야 함

---

## 5. 금감원 검토 및 조치 (제36조 제3항)

### 5.1 검토 권한

금융감독원장은 결과보고서 검토 후:
- 보안수준이 **충분하지 않다고 인정**되는 경우
- **개선 및 보완 요구** 가능

### 5.2 후속 조치

```yaml
보안수준_충분:
  조치: 서비스 출시 진행

보안수준_미흡:
  조치:
    - 금감원 개선/보완 요구 수령
    - 요구사항에 따른 보완 조치 이행
    - 보완 결과 재제출 (필요 시)
    - 승인 후 서비스 출시
```

## 6. 주요 확인사항 (체크리스트)

### 6.1 심의 대상 여부

- [ ] 정보통신망을 이용하는가?
- [ ] 이용자를 대상으로 하는 서비스인가?
- [ ] 신규 전자금융업무에 해당하는가?
- [ ] 복수 기관 공동 표준 제정에 해당하는가?

### 6.2 심의 절차

- [ ] CISO 주관으로 심의를 실시하였는가?
- [ ] 금감원장이 정하는 기준과 절차를 따랐는가?
- [ ] 정보보호위원회 심의가 필요한 사항인가?

### 6.3 결과보고서 제출

- [ ] 심의 완료 후 7일 이내 제출하였는가?
- [ ] 제출 면제 조건에 해당하는가? (1년 이내 사고 미발생 + 금감원 기준)
- [ ] 제출 면제 기관에 해당하는가?

### 6.4 금감원 검토 대응

- [ ] 개선/보완 요구 수령 시 대응 계획이 있는가?
- [ ] 보완 조치 완료 후 재제출 절차가 준비되어 있는가?

## 7. ISMS-P와의 차이

| 항목 | EFSR 제36조 | ISMS-P |
|------|-------------|--------|
| **심의 주체** | 금융회사 자체 (CISO) | - |
| **심의 대상** | 신규 전자금융업무, 공동 표준 | - |
| **외부 보고** | 금감원 제출 의무 (7일 이내) | 해당 없음 |
| **감독 기관 검토** | 금감원 검토 및 개선 요구 | 해당 없음 |
| **법적 구속력** | 법적 의무 | - |

**핵심 차이**: ISMS-P에는 신규 서비스 출시 전 **사전 심의 및 감독기관 보고 절차가 없음**. 자체 보안성심의는 금융권 고유의 사전 통제 장치.

## 8. 실무 적용 가이드

### 8.1 심의 시점

```
개발 완료 → 테스트 완료 → 보안성심의 → (7일 내) 금감원 제출 → 서비스 출시
                          ↑
                     출시 전 필수 단계
```

### 8.2 심의 범위 예시

| 서비스 유형 | 심의 포인트 |
|-------------|------------|
| 모바일뱅킹 신기능 | 인증 방식, 암호화, 접근통제, 이상거래탐지 |
| 간편결제 연동 | 본인확인, 거래정보 보호, 제3자 연동 보안 |
| 오픈API 제공 | API 인증, 권한 관리, 로깅, 이용기관 통제 |
| 비대면 계좌개설 | 본인확인 수단, 신원정보 보호, 부정개설 방지 |

### 8.3 결과보고서 구성 (권장)

1. **심의 개요**: 대상 서비스, 심의 일자, 참여자
2. **서비스 설명**: 기능, 아키텍처, 데이터 흐름
3. **보안 요건 분석**: 식별된 위험, 적용 통제
4. **심의 결과**: 보안 적합성 판단, 조건부 승인 사항
5. **후속 조치 계획**: 미결 사항, 개선 일정
