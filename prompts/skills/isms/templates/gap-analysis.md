# GAP 분석

## 개요
현재 보안 수준과 ISMS-P 인증기준 간의 차이(Gap)를 식별하고 조치 계획을 수립합니다.

## GAP 분석 방법론

### 1단계: 현황 파악
- 기존 정책/절차 문서 검토
- 시스템 구성 현황 파악
- 담당자 인터뷰

### 2단계: 기준 대비 평가
각 항목별 이행 수준 평가:
- ◯ 충족: 기준 완전 충족
- △ 부분 충족: 일부 미흡
- × 미충족: 미이행 또는 대폭 개선 필요
- N/A: 해당 없음

### 3단계: 조치 계획 수립
우선순위:
- P1: 즉시 조치 (법적 위반 위험)
- P2: 단기 조치 (심사 전 완료)
- P3: 중기 조치 (점진적 개선)

## 평가 기준 상세

### 충족 (◯) 조건
- 관련 정책/절차 문서화
- 실제 운영 증적 확보
- 최근 1년 이내 검토/갱신

### 부분 충족 (△) 조건
- 문서는 있으나 운영 증적 부족
- 일부 항목만 이행
- 최신화 필요

### 미충족 (×) 조건
- 정책/절차 미수립
- 운영 증적 전무
- 기준 미인지
