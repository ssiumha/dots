# /jd-health

문서 시스템 건강도를 체크합니다.

## CLI 권장

컨텍스트 절약을 위해 CLI 사용 권장:

```bash
jd health          # 건강도 체크
jd health --fix    # 자동 수정
```

CLI 미설치 시 아래 워크플로우로 진행합니다.

---

## 사용법

```
/jd-health
/jd-health --fix
```

## 검사 항목

| 항목 | 설명 | 자동 수정 |
|------|------|----------|
| JDex 동기화 | 파일 ↔ JDex 일치 | ✅ |
| ID 중복 | 카테고리 내 ID 유일성 | ❌ |
| 필수 필드 | frontmatter 필수 항목 | ✅ (기본값) |
| 링크 유효성 | 문서 간 참조 검증 | ❌ |
| 오래된 문서 | 180일+ 미수정 | ❌ |
| deprecated | 폐기 예정 문서 정리 | ❌ |

## 출력 형식

```
📊 문서 건강도: 85/100

✅ 양호 (15):
- JDex 동기화 완료
- ID 중복 없음

⚠️ 주의 필요 (3):
- 21.01: 180일 이상 미수정
- 31.02 → [[22.03]]: 링크 깨짐
- 63.01: frontmatter 누락 (severity)

❌ 조치 필요 (1):
- 21.02: deprecated 상태, 아카이브 권장

🔧 권장 조치:
1. 21.01 검토 후 상태 업데이트
2. 31.02 링크 수정 (22.03 → ?)
3. 63.01 severity 필드 추가
4. 21.02 → 90-99-Archive로 이동
```

## --fix 옵션

자동 수정 가능한 항목을 수정합니다.

```
/jd-health --fix
```

**자동 수정 항목:**
- JDex 동기화 (누락 항목 추가)
- 기본 frontmatter 추가 (id, title, status, date)
- 파일명 정규화 (slug 형식)

**수정 결과:**
```
🔧 자동 수정 완료

수정됨:
- JDex: +2 항목 추가
- 31.03: frontmatter 추가

수동 조치 필요:
- 21.01: 상태 검토 필요
- 31.02: 링크 수정 필요
```
