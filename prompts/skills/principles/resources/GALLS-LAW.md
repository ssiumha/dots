# GALLS-LAW — Gall's Law

## 정의

> "A complex system that works is invariably found to have evolved from a simple system that worked. A complex system designed from scratch never works and cannot be patched up to make it work."

처음부터 복잡하게 설계한 시스템은 작동하지 않고, 고칠 수도 없다. 단순한 시스템부터 만들어서 작동을 확인하고, 거기서 점진적으로 확장해야 한다.

## 핵심 판단

- **"이 시스템이 단순한 버전에서 출발했는가?"** — 아니면 위험
- **"현재 작동하는 상태를 유지하면서 확장하고 있는가?"** — 아니면 위험
- **"처음부터 모든 케이스를 커버하려 하는가?"** — 그렇다면 위반

## 위반 신호

### 기계적 검증 (자동화 가능)

| 신호 | 검증 방법 |
|------|-----------|
| 첫 커밋부터 파일 수 과다 (50+) | 초기 커밋의 파일 수 확인 |
| 추상화 레이어가 사용처보다 먼저 존재 | 인터페이스는 있으나 구현체가 1개 |
| 설정/config 파일이 코드보다 먼저 복잡 | 설정 항목 수 vs 실제 사용 비율 |
| 테스트 없이 거대한 초기 구조 | 첫 PR/커밋의 테스트 커버리지 |

### 판단 필요 (사람/LLM)

| 신호 | 설명 |
|------|------|
| "나중에 필요할 테니 미리 만들어두자" | YAGNI + Gall's Law 동시 위반 |
| 아직 사용자가 없는데 마이크로서비스 10개 | 검증 안 된 복잡성 |
| "아키텍처 먼저 잡고 시작하자" (코드 없이) | 작동하는 단순 시스템 부재 |
| 6개월 개발 후 첫 배포에서 전면 장애 | Big Bang 출시 |
| 프레임워크/인프라에 시간을 다 쓰고 기능이 없음 | 수단과 목적 전도 |

## 스코어링

| 등급 | 기준 |
|------|------|
| **PASS** | 단순한 작동 버전에서 출발, 점진적 확장, 각 단계가 작동 |
| **WARN** | 일부 과도한 사전 설계 존재하나 핵심 기능은 먼저 작동 |
| **FAIL** | 전체 구조를 한 번에 설계/구현 OR 작동 확인 없이 복잡성 증가 OR Big Bang 출시 |

## 점진적 진화 경로

```
단순한 작동 시스템
  → 피드백으로 경계 발견
    → 필요한 곳에만 복잡성 추가
      → 다시 작동 확인
        → 반복

❌ 절대 안 되는 경로:
전체 설계 문서 → 전체 구현 → 전체 테스트 → Big Bang 출시 → 장애
```

## 개선 패턴

| 상황 | 적용 |
|------|------|
| 처음부터 마이크로서비스 | **모놀리스 → 모듈 분리 → 필요 시 서비스 분리** |
| 처음부터 범용 프레임워크 | **구체적 문제 해결 → 패턴 발견 → 추상화** |
| 처음부터 완벽한 스키마 | **MVP 스키마 → 마이그레이션으로 진화** |
| 처음부터 전체 아키텍처 | **Walking Skeleton** — 최소 기능이 끝까지 관통하는 뼈대 먼저 |

## 다른 원칙과의 관계

| 원칙 | 관계 |
|------|------|
| YAGNI | YAGNI는 "불필요한 것을 안 만든다", Gall's Law는 "필요해도 단순하게 시작한다" |
| KISS | KISS는 해법의 단순성, Gall's Law는 시스템 진화 전략의 단순성 |
| ROT | 추상화 타이밍 — 3번 반복 후 추상화 = Gall's Law의 점진적 진화와 일치 |
| BC | 모놀리스에서 경계가 명확해지면 그때 분리 — Gall's Law의 실천 |

## 주의

- Gall's Law ≠ "설계하지 마라". 방향은 정하되 구현은 점진적으로
- "단순한 시스템"은 "대충 만든 시스템"이 아니다. 작지만 올바르게 작동하는 시스템
- 이미 복잡한 시스템을 리팩토링할 때도 적용: 한 번에 전체 교체가 아니라 Strangler Fig 패턴으로 점진적 교체
- 규제/보안 요건이 있는 도메인에서는 최소 요건이 이미 복잡할 수 있다. 그래도 "가장 단순한 준수 버전"부터 시작
