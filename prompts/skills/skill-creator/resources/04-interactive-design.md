# Interactive Skill Design

대화형 skill 설계 패턴입니다.

## 핵심 원칙

Skill을 **대화**처럼 설계하세요:
- 한 번에 모든 정보 요구 X
- Phase별로 단계적 진행
- 사용자 입력에 따라 분기

## Phase 패턴

### 선택형 Phase

```markdown
## Phase 1: Type Selection

어떤 유형이 필요한가요?

1. 유형 A - 설명
2. 유형 B - 설명
3. 유형 C - 설명

번호로 답변해주세요.
```

사용자가 "2"를 입력하면 해당 유형 워크플로우로 진행.

### 정보 수집 Phase

```markdown
## Phase 2: Context Gathering

다음 질문에 답변해주세요:

1. 대상은 누구인가요?
2. 목표는 무엇인가요?
3. 제약사항이 있나요?

다 입력하셨으면 "continue"를 입력해주세요.
```

### 확인 Phase

```markdown
## Phase 3: Confirmation

다음 내용으로 진행할까요?

- 유형: {선택된 유형}
- 대상: {입력된 대상}
- 목표: {입력된 목표}

맞으면 "확인", 수정이 필요하면 "수정"을 입력해주세요.
```

## Continue 패턴

긴 분석이나 복잡한 프레임워크에서 사용:

```markdown
## Module 1: Problem Analysis

[분석 내용]

---
다음 모듈로 진행하려면 "continue"를 입력하세요.
```

장점:
- 사용자가 진행 속도 조절
- 각 단계 결과 검토 가능
- 필요시 중단/수정 가능

## 분기 처리

```markdown
## Phase 1: Initial Assessment

프로젝트 규모는?

1. 소규모 (1-3명, 1개월 이내)
2. 중규모 (4-10명, 3개월 이내)
3. 대규모 (10명+, 6개월+)

[사용자 선택에 따라]

### 소규모 선택 시
→ 간단한 구조 권장
→ 워크플로우 A로 진행

### 대규모 선택 시
→ 상세 계획 필요
→ 추가 질문 후 워크플로우 B로 진행
```

## 예시: Email Composer Skill

```markdown
---
name: email-composer
description: 이메일 작성 시 사용. 상황에 맞는 톤과 구조.
---

## Phase 1: Context

누구에게 보내는 이메일인가요?

1. 동료/팀원
2. 상사/경영진
3. 클라이언트
4. 콜드 아웃리치

## Phase 2: Purpose

이메일의 목적은?

1. 정보 전달
2. 요청/부탁
3. 피드백 제공
4. 사과/해명
5. 축하/감사

## Phase 3: Draft

[선택에 따른 템플릿 적용]

초안입니다. 수정이 필요하면 말씀해주세요.
```

## 안티패턴

| ❌ 피하기 | ✅ 권장 |
|----------|--------|
| 한 번에 10개 질문 | Phase별 2-3개 질문 |
| 긴 설명 후 질문 | 간결한 설명 + 명확한 질문 |
| 모호한 선택지 | 번호 + 짧은 설명 |
| 무한 대기 | "continue" 또는 번호로 명시적 진행 |

## 적용 시나리오

Interactive 설계가 효과적인 경우:
- 결과물이 사용자 입력에 크게 의존
- 여러 유형/변형이 존재
- 복잡한 분석 프레임워크
- 단계별 검토가 필요한 작업
