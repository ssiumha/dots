# Theory (Part III)

Kent Beck "Tidy First?" Part III — 정리의 **이론적 근거**.

## Structure vs Behavior

소프트웨어는 두 가지 방식으로 가치를 전달한다:

| 구분 | 행동 (Behavior) | 구조 (Structure) |
|------|-----------------|------------------|
| 정의 | 소프트웨어가 **하는 것** | 소프트웨어가 **되어 있는 방식** |
| 가치 | 현재 사용자에게 직접 가치 | 미래 변경을 가능하게 함 |
| 변경 | 테스트 결과가 바뀜 | 테스트 결과 불변 |
| 예시 | 기능 추가, 버그 수정 | 정리, 리팩토링 |

- 행동만 가치 있다고 착각하기 쉬움
- 구조가 나쁘면 행동 변경 비용이 기하급수적으로 증가
- **정리는 구조에 투자하여 미래 행동 변경 비용을 낮추는 행위**

## Economics: 화폐의 시간가치 vs 옵션 가치

### 화폐의 시간가치 (Time Value of Money)

- 오늘의 1원 > 내일의 1원
- 수익은 당기고, 비용은 뒤로 미루라
- **이 논리만 따르면**: 정리하지 말고 기능부터 → 기술 부채 누적

### 옵션 가치 (Option Value)

- 좋은 구조는 **옵션**을 만든다 — 미래에 할 수 있는 것의 범위를 넓힘
- 정리는 비용이지만, 결과로 생긴 옵션의 가치가 비용을 초과할 수 있음
- 불확실성이 클수록 옵션 가치가 높아짐
- **정리의 경제적 정당화**: 당장 비용이지만, 미래 변경 옵션의 가치 > 정리 비용

### 균형

```
정리 먼저 (First)  ← 옵션 가치가 높을 때
정리 나중 (Later)  ← 시간가치가 더 중요할 때 (긴급 배포 등)
정리 안함 (Never)  ← 옵션 가치 ≈ 0 (변경 예정 없음)
```

## Coupling (결합도)

**두 요소가 결합되어 있다** = 하나를 변경하면 다른 것도 변경해야 함.

### 결합의 속성

- **정도**: 하나가 바뀔 때 다른 쪽이 얼마나 바뀌는가
- **범위**: 연쇄적으로 몇 개의 요소가 영향받는가
- **비용**: 실제로 변경이 발생하는 경우에만 비용이 됨

### 핵심 인사이트

- 결합 자체가 나쁜 게 아니라, **실제로 변경되는 방향**의 결합이 비용
- 이론적 결합이 아닌, 실제 변경 패턴 기준으로 판단
- 결합을 완전히 제거할 수 없음 — 관리하는 것

## Cohesion (응집도)

**함께 변경되는 요소는 함께 놓아야 한다.**

- 응집도 높음 = 하나의 변경이 하나의 모듈 안에서 완결
- 응집도 낮음 = 하나의 변경을 위해 여러 모듈을 수정

### 결합도와 응집도의 관계

```
응집도 ↑  →  변경이 한 곳에 집중  →  결합 비용 ↓
응집도 ↓  →  변경이 분산          →  결합 비용 ↑
```

- Cohesion Order, Chunk Statements 패턴이 응집도를 높이는 tidying

## Constantine's Equivalence

> **cost(software) ≈ cost(change) ≈ cost(big changes) ≈ coupling**

Larry Constantine의 등가 원리:

1. 소프트웨어 비용 ≈ 변경 비용 (유지보수가 전체 비용의 대부분)
2. 변경 비용 ≈ 큰 변경의 비용 (파레토: 소수의 큰 변경이 대부분의 비용)
3. 큰 변경의 비용 ≈ 결합도 (결합이 변경을 연쇄시켜 크게 만듦)

**따라서**: 결합도를 줄이면 소프트웨어 비용이 줄어든다.
**정리의 궁극적 목표**: 불필요한 결합을 제거하여 변경 비용을 낮추는 것.

## Reversible Structure Changes

> "대부분의 소프트웨어 설계 결정은 쉽게 되돌릴 수 있다."

- 가역적 결정: Guard Clause, Rename, Extract Helper → 빠르게 시도
- 비가역적 결정: API 공개, DB 스키마 변경, 프로토콜 변경 → 신중하게
- **Tidying은 대부분 가역적** — 부담 없이 시도하고, 안 맞으면 되돌린다
- 가역적 결정에 비가역적 결정의 프로세스를 적용하지 말 것 (과도한 리뷰, 승인 등)
