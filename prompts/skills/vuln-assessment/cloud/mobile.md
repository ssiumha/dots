# 이동통신 취약점 점검항목

> 출처: KISA 주요정보통신기반시설 기술적 취약점 분석평가 방법 상세가이드 (2026)
> 총 항목: 4개 (M-01 ~ M-04)

---

### M-01 이동통신망 엣지 네트워크 운영 시 인터넷 연결 등을 고려한 보안기술을 적용

점검ID: M-01 | 위험도: 상 | 카테고리: 운영 관리

**점검내용**: 이동통신망 관리 네트워크에 보안기술을 사용한 안전 분리 운영 여부 점검

**판단기준**:
- ✅ 양호: 이동통신망 엣지 네트워크 운영 시 인터넷 연결 등을 고려하여 보안기술을 적용하고 관리 네트워크를 안전하게 분리 운영하는 경우
- ❌ 취약: 이동통신망 관리 네트워크에 보안기술이 적용되어 있지 않고 분리 운영하지 않는 경우

**점검방법**:
1. 엣지 네트워크 관리자 네트워크 접근 권한 관리 확인
   - 접근 권한 사용자 최소화 여부
   - 1인 1계정 원칙 준수 여부
   - 접속자, 접근 권한, 접근 목적, 접근 일시 및 주기 문서화 여부

2. 인증 및 로깅 설정 확인
   - Token을 포함한 다중 인증 적용 여부
   - 접속 성공/실패 로그 최소 1년 이상 저장 여부

3. 네트워크 분리 확인
   - 관리 네트워크와 단말 사용자 트래픽 네트워크 간 연결 지점 부재 확인
   - 논리적 네트워크 분리 확인
   - 상호 다른 네트워크 주소 체계 사용 확인
   - 최대 비트 암호화 적용 확인
   - 엣지 내 저장 장치 분리 및 디스크 암호화 확인

4. 이상징후 탐지 및 알림 확인
   - 다수의 접근시도/실패 실시간 통보 여부
   - 브로드캐스팅 통신, 디스커버리 통신 탐지 및 통보 여부
   - 관리 네트워크 접근/작업/권한에 대한 가시성 확보 여부

5. 보안 감사 체계 확인
   - 직무 분리된 보안담당자에 의한 주기적 보안 검토 및 감사 시행 여부

**조치방법**:
1. 엣지 네트워크 관리자 계정 관리 강화
   - 접근 권한 사용자를 최소화하고 1인 1계정 원칙 적용
   - 접속자, 접근 권한, 접근 목적, 접근 일시 및 주기를 사전에 문서로 정의

2. 다중 인증 및 로깅 구현
   - Token을 포함한 다중 인증 시스템 구축
   - 접속 성공/실패 로그를 최소 1년 이상 보관하도록 설정

3. 네트워크 안전 분리
   - 관리 네트워크와 트래픽 네트워크 간 물리적/논리적 분리
   - 서로 다른 네트워크 주소 체계 사용
   - 최대 비트 암호화 기술 적용
   - 엣지 저장 장치 분리 및 디스크 암호화

4. 이상징후 탐지 시스템 구축
   - 다수의 접근시도, 접근 실패, 브로드캐스팅/디스커버리 통신 실시간 탐지 및 통보 체계 구축
   - 관리 네트워크 접근, 작업, 권한 활동에 대한 가시성 확보

5. 정기적 보안 감사 수행
   - 관리자와 직무 분리가 명확한 보안담당자 지정
   - 주기적인 보안 검토 및 감사 시행

### M-02 이동통신망에서 가상화 기술 적용 시 계정 관리, 이상징후 탐지 등 보안 설정 또는 기술 적용

점검ID: M-02 | 위험도: 상 | 카테고리: 운영 관리

**점검내용**: 물리적 계층, 가상화 계층, 애플리케이션 계층 등의 수직적 계층에 적합한 보안기술 적용 여부 점검

**판단기준**:
- ✅ 양호: 이동통신망에서 가상화 기술 적용 시 각 계층에 적합한 계정 관리, 이상징후 탐지 등 보안 설정 또는 기술을 적용하고 수직적 보안을 고려하는 경우
- ❌ 취약: 각 계층에 적합한 보안기술이 적용되어 있지 않은 경우

**점검방법**:
1. 3GPP 표준 보안 설정 적용 확인
   - 가상화 플랫폼 컨트롤 영역 보안 설정 확인
   - 사용자 데이터 영역 보안 설정 확인
   - 3GPP 표준 필수 보안 설정 최대 적용 여부

2. 계정 및 권한 관리 정책 확인
   - 계층별 권한에 적합한 사용자/그룹별 정의 여부
   - 사전 정의된 변경 절차 존재 여부
   - 변경 절차 적합한 실행 구현 여부

3. IAM 계정 이상징후 탐지 확인
   - 사전 정의된 절차 외 변경 시 알람/경고 설정 여부
   - 변경 내용 로그 기록 및 주기적 검토 여부

4. SDN/NFV 통합 관리 확인
   - SDN/NFV의 플랫폼 IAM 통합 여부
   - 미통합 서비스의 로깅, 알림, 감사 기능 적용 여부
   - 서비스 슬라이싱별 최소 보안성 유지 여부

5. 계층별 보안 관제 기능 확인
   - 계층 침입 탐지 기능
   - 권한 탈취 식별 기능
   - 데이터 유출방지 기능
   - 네트워크 슬라이싱 간 격리, 보호 및 모니터링
   - IoT 기기 접근통제 기능
   - 비정상 행위 식별 기능

**조치방법**:
1. 3GPP 표준 보안 설정 적용
   - 가상화 플랫폼에서 컨트롤 영역과 사용자 데이터 영역별로 3GPP 표준 필수 보안 설정 최대 적용

2. 계정 및 권한 관리 체계 구축
   - 플랫폼 사용자 계정 및 권한 관리 보안 정책을 사전 정의
   - 계층별 권한에 적합하도록 사용자/그룹별 정의 및 부여
   - 보안담당자가 필요에 따라 변경 절차를 모두 정의하고 적합하게 실행되도록 구현

3. IAM 계정 모니터링 강화
   - 사전 정의된 변경 절차 외 변경 발생 시 알람 및 경고 기능 설정
   - 변경 내용 로그 기록 및 주기적 검토 체계 구축

4. SDN/NFV 통합 관리
   - SDN/NFV를 플랫폼 IAM에 통합
   - 기술적 여건으로 미통합 시 로깅, 알림, 감사 기능을 추가 적용
   - 각 서비스 슬라이싱별 최소 보안성 유지

5. 계층별 보안 관제 체계 구축
   - 계층 침입 탐지, 권한 탈취 식별, 데이터 유출방지 기능 구현
   - 네트워크 슬라이싱 간 격리, 보호 및 모니터링 체계 구축
   - IoT 기기 접근통제 및 비정상 행위 식별 기능 적용

### M-03 이동통신망 장비 및 SW 제조사 등과 보안 정책을 마련하여 운영

점검ID: M-03 | 위험도: 상 | 카테고리: 운영 관리

**점검내용**: 일정 수준의 보안 요구 수준 및 운영 계획과 수명주기를 반영하여 보안 정책 운영 여부 점검

**판단기준**:
- ✅ 양호: 이동통신망 장비 및 SW 제조사 등과 보안 정책을 마련하여 운영 계획과 수명주기를 반영하고 일정 수준의 보안 요구 수준을 유지하는 경우
- ❌ 취약: 운영 계획과 수명주기를 반영하지 않고 보안 요구 수준이 낮은 환경으로 서비스를 운영하는 경우

**점검방법**:
1. 네트워크 보안 수준 기준 확인
   - 사용자 장치부터 코어 네트워크까지 종단 경로 보안 수준 유지 여부
   - 네트워크 설계 프로세스 초기 보안 고려 여부
   - 최저 보안 기준 수립 여부

2. SLA(Service Level Agreement) 확인
   - 단말, 액세스 네트워크, 코어 네트워크 전체 물리적 도메인 보안 수준 정의 여부
   - SLA 작성 및 보안 수준 일정 수준 유지 명시 여부

3. 보안 표준안 확인
   - 필수적인 보안 항목 정의 여부
   - 보안 수준 개선 가능 항목 포함 여부
   - 개발 주기 동안 최신 잠재적 위험 개선 고려 여부
   - 피해 발생 회피 및 피해 최소화 고려 여부

4. 보안 사전 테스트 및 검증 확인
   - 운영 환경 적용 전 보안 사전 테스트 수행 여부
   - 설계 시 예측하지 못한 운영상 취약점 식별 및 대응 여부
   - 구축 완료 후 주기적 위험성 검토 및 보안 개선 활동 계획/실행 여부

5. 저보안 단말 및 IoT 장비 관리 확인
   - 제3자 서비스 사업자 API 보안 관제 여부
   - IoT 운영 및 원격 관리 보안 관제 여부
   - 자동 보안 관제 장치 기술 적용 고려 여부

**조치방법**:
1. 네트워크 보안 수준 기준 수립
   - 사용자 장치부터 코어 네트워크까지 종단 경로 보안 수준 유지
   - 네트워크 설계 프로세스 초기 단계에서 보안 고려
   - 최저 보안 기준 수립 및 문서화

2. SLA 작성 및 적용
   - 단말, 액세스 네트워크, 코어 네트워크 전체 물리적 도메인에 걸친 SLA 작성
   - 보안 수준이 일정 수준으로 유지되도록 명시

3. 보안 표준안 정의
   - 장비 도입 시 필수 보안 항목과 개선 가능 항목 포함하여 보안 표준안 정의
   - 개발 주기 동안 최신 잠재적 위험 개선
   - 피해 발생 회피 및 피해 최소화 고려

4. 보안 사전 테스트 및 주기적 검증
   - 운영 환경 적용 전 보안 사전 테스트 및 검증 수행
   - 설계 시 예측하지 못한 운영상 취약점 식별 및 대응
   - 구축 완료 후에도 주기적으로 위험성 검토 및 보안 개선 활동 계획 및 실행

5. 저보안 단말 및 IoT 장비 보안 강화
   - 보안 요구 수준이 낮은 종단 사용자 단말기 및 IoT 장비에 대해 추가 보안 관제
   - 제3자 서비스 사업자 API, IoT 운영 및 원격 관리에 자동 보안 관제 장치 기술 적용

### M-04 이동통신망 장비 구축과 네트워크 설계 시 보안 내재화를 수행

점검ID: M-04 | 위험도: 상 | 카테고리: 운영 관리

**점검내용**: 보안담당자 지정 및 보안 업무 인력이 개통 운영 참여 여부 점검

**판단기준**:
- ✅ 양호: 이동통신망 장비 구축과 네트워크 설계 시 보안담당자를 지정하고, 보안 업무 인력이 초기 서비스 설계부터 개통 운영까지 참여하며 관련 내용을 주기적으로 정보보호 책임자에게 보고 및 문서화하는 경우
- ❌ 취약: 보안담당자를 지정하지 않고, 보안 업무 인력이 개통 운영에 참여하지 않는 경우

**점검방법**:
1. 국제표준 보안 요건 준수 확인
   - 서비스 시작 전 도입 대상 장비의 국제표준 보안 요건 준수 여부
   - 서비스 제공자와 사전 검토 및 확인 여부

2. 보안담당자 참여 확인
   - 장비 구축 시 보안담당자 입회 여부
   - 종단 간 보안(End to End Security) 관점의 전 도메인 영향도 분석 여부

3. 네트워크 아키텍처 분리 설계 확인
   - 산업용 제어 네트워크 분리 여부
   - 내부 업무용 네트워크 분리 여부
   - 인터넷 접근 가능한 사용자 네트워크 분리 여부
   - 개별 네트워크 구성 또는 네트워크 슬라이싱 분리 사용 여부

4. 이상징후 탐지 및 알림 확인
   - 엣지 네트워크 다수의 접근시도/실패 실시간 통보 여부
   - 브로드캐스팅/디스커버리 통신 탐지 및 통보 여부

5. 관리 네트워크 가시성 및 감사 확인
   - 관리 네트워크 접근, 작업, 권한 및 활동에 대한 가시성 확보 여부
   - 이상징후 탐지 시스템 구축 여부
   - 직무 분리된 보안담당자에 의한 주기적 보안 검토 및 감사 여부

**조치방법**:
1. 국제표준 보안 요건 준수
   - 서비스 시작 전 도입 대상 장비가 국제표준 보안 요건을 준수하는지 서비스 제공자와 사전 검토 및 확인

2. 보안담당자 지정 및 참여
   - 초기 장비 구축 전부터 보안담당자 지정
   - 서비스 설계, 장비 선정, 아키텍처 구성, 디자인, 검토 심의 등 생명 주기 전반에 보안담당자 참여
   - 장비 구축 시 보안담당자 입회 진행
   - 종단 간 보안(End to End Security) 관점으로 전 도메인 영향도 분석

3. 네트워크 아키텍처 분리 설계
   - 산업용 제어 네트워크, 내부 업무용 네트워크, 인터넷 접근 가능한 사용자 네트워크를 각각 분리
   - 개별 네트워크로 구성하거나 네트워크 슬라이싱 분리를 사용하여 독립적으로 분리 구성

4. 이상징후 탐지 및 알림 체계 구축
   - 엣지 네트워크 관리자 계정의 다수 접근시도, 다수 접근 실패, 브로드캐스팅/디스커버리 통신을 실시간으로 보안담당자에게 통보 및 검토

5. 관리 네트워크 가시성 확보 및 정기 감사
   - 관리 네트워크 접근, 작업, 시스템 권한 및 활동에 대한 가시성 확보
   - 이상징후 탐지 시스템 구축
   - 관리자와 직무 분리가 명확한 보안담당자에 의한 주기적 보안 검토 및 감사 시행
   - 관련 내용을 주기적으로 정보보호 책임자에게 보고 및 문서화
