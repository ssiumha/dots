# 보안장비 취약점 점검항목

> 출처: KISA 주요정보통신기반시설 기술적 취약점 분석평가 방법 상세가이드 (2026)
> 총 항목: 23개 (S-01 ~ S-23)

## 1. 계정 관리

### S-01 보안장비 Default 계정 변경

> 점검ID: S-01 | 위험도: 상 | 카테고리: 계정관리

**점검내용**: 보안 장비에 기본적으로 설정된 관리자 계정의 변경 여부 점검

**판단기준**:
- ✅ 양호: 장비에서 제공하고 있는 기본 계정을 변경하여 사용하는 경우 (기본 계정 변경이 불가능할 경우 기본 비밀번호 변경으로 보완 필요)
- ❌ 취약: 장비에서 제공하고 있는 기본 계정을 변경할 수 있으나 변경하지 않고 사용하는 경우

**점검방법**:
1. Web을 통한 접속
2. 기본 계정, 비밀번호 입력하여 접속 확인
3. 보안 장비에서 제공하고 있는 계정 메뉴에서 기본 계정 확인

**조치방법**:
1. 보안 장비에서 제공하고 있는 계정 메뉴에서 기본 계정 변경
2. 기본 계정 변경이 불가능할 경우 기본 비밀번호 변경으로 보완

**참고사항**:
- 기본 계정: 장비 제조업체에서 출고 시 설정되어 나오는 기본 관리자 계정정보
- 보안 위협: 비인가자가 공개된 기본 관리자 계정정보를 통하여 보안 장비에 불법적인 접근 및 보안 장비 설정값을 변경함으로써 시스템 침입 경로 제공 및 보안 장비를 무력화할 수 있는 위험이 존재함

---

### S-02 비밀번호 관리정책 설정

> 점검ID: S-02 | 위험도: 상 | 카테고리: 계정관리

**점검내용**: 보안 장비의 계정 비밀번호가 기관 정책에서 정의된 비밀번호 관리 정책 적용 여부 점검

**판단기준**:
- ✅ 양호: 비밀번호 관리 정책에 맞는 비밀번호가 사용된 경우
- ❌ 취약: 비밀번호 관리 정책에 맞지 않는 비밀번호가 사용된 경우

**점검방법**:
1. Web을 통한 접속
2. 기본 계정, 비밀번호 입력 후 접속 확인
3. 비밀번호 메뉴에서 비밀번호 정책 확인

**조치방법**:
1. 해당 기관의 비밀번호 관리 정책에 따라 적합하게 설정
2. 보안 장비가 제공하는 범위에서 비밀번호 관리 정책에 맞는 비밀번호로 변경
3. 복잡성 및 최소 길이: 특수문자, 숫자, 영 소문자 포함 8자리 이상, 최소 사용기간(1일) 설정

**참고사항**:
- 비밀번호 관리 정책: 기관 정책에서 정의된 비밀번호 복잡성, 비밀번호 최소 길이, 비밀번호 최소 사용기간, 비밀번호 최대 사용기간
- 보안 위협: 비밀번호 관리 정책에 맞지 않는 비밀번호를 사용할 경우, 비인가자의 비밀번호 추측 공격(무차별 대입 공격, 사전 대입 공격 등)에 계정 비밀번호가 유출되어 비인가자가 보안 장비에 접근할 위험이 존재함

---

### S-03 보안장비 계정별 권한 설정

> 점검ID: S-03 | 위험도: 상 | 카테고리: 계정관리

**점검내용**: 보안 장비에 등록한 계정들에 대해 업무에 불필요한 권한 여부 점검

**판단기준**:
- ✅ 양호: 사용자별 계정의 용도 파악 및 적절한 권한이 부여된 경우
- ❌ 취약: 사용자별 계정의 용도 파악 및 적절한 권한이 부여되지 않은 경우

**점검방법**:
1. 보안 장비에서 제공하고 있는 계정 메뉴에서 계정별 권한 확인

**조치방법**:
1. 보안 장비에서 제공하고 있는 계정 메뉴에서 기존 계정의 권한 검토 (불필요한 권한 삭제)
2. 단일 계정을 여러 사용자가 공유 시 사용자별 계정 생성 및 권한 차등 부여
3. 사용자별 계정의 용도 파악 및 적절한 권한 부여

**참고사항**:
- 관리자 권한은 최소한의 계정에만 부여
- 보안 위협: 사용자별 계정의 용도 파악 및 적절한 권한을 부여하지 않을 경우, 권한 없는 사용자의 의도하지 않은 보안 정책 수정이나 보안 장비 설정값 변경을 통하여 비인가자에게 시스템 침입 경로를 제공할 위험이 존재함

---

### S-04 보안장비 계정 관리

> 점검ID: S-04 | 위험도: 상 | 카테고리: 계정관리

**점검내용**: 보안 장비에 등록된 계정 중 사용하지 않는 계정을 제거 또는 관리 여부 점검

**판단기준**:
- ✅ 양호: 불필요한 계정을 제거하거나 관리된 경우
- ❌ 취약: 불필요한 계정을 제거하지 않거나 관리되지 않은 경우

**점검방법**:
1. 보안 장비에서 제공하고 있는 계정 메뉴에서 계정 및 담당자 확인

**조치방법**:
1. 사용하지 않는 계정 삭제
2. 불필요한 공용 계정 및 휴면계정 제거
3. 공용 계정 사용 시 사용자별 계정 생성 및 시스템 접근 이력을 관리하여 책임 추적성 확보

**참고사항**:
- 공용 계정 및 휴면계정의 계정 탈취 시 침해사고 발생 후 사후 추적이 어려움
- 시스템이 아닌 사람을 중심으로 하는 통합된 계정 관리가 중요함
- 보안 위협: 불필요한 계정을 제거하지 않거나 관리하지 않을 경우, 비인가자의 공격을 통해 불필요한 계정 탈취의 위험이 존재함

---

### S-05 계정 잠금 임계값 설정

> 점검ID: S-05 | 위험도: 상 | 카테고리: 계정관리

**점검내용**: 보안 장비에서 제공하고 있는 로그인 실패 임계값 설정 적용 여부 점검

**판단기준**:
- ✅ 양호: 로그인 실패 임계값을 5회 이하로 설정된 경우
- ❌ 취약: 로그인 실패 임계값을 5회 이하로 설정되지 않은 경우

**점검방법**:
1. 보안 장비에서 제공하고 있는 계정 메뉴에서 로그인 실패 임계값 확인

**조치방법**:
1. 보안 장비에서 제공하고 있는 계정 메뉴에서 로그인 실패 임계값을 5회 이하로 설정
2. 로그인 실패 임계값을 5회 이하로 제한

**참고사항**:
- 기종에 따라 Limited/Lockout로 표시됨
- 보안 장비에 계정 잠금 시간 기능을 지원할 시 함께 설정하면 보안성이 향상됨
- 보안 위협: 로그인 실패 임계값 설정을 적용하지 않을 경우, 비인가자는 자동화된 방법을 통하여 모든 계정에 대해 비밀번호 추측 공격을 자유롭게 시도하여 비밀번호 정보가 유출될 위험이 존재함

---

## 2. 접근 관리

### S-06 보안장비 원격 관리 접근 통제

> 점검ID: S-06 | 위험도: 상 | 카테고리: 접근관리

**점검내용**: 보안 장비 원격 관리 시 관리자 IP 또는 특정 IP만 접근할 수 있도록 설정하였는지 점검

**판단기준**:
- ✅ 양호: 원격 관리 시 관리자 IP 또는 특정 IP만 접근할 수 있도록 설정된 경우
- ❌ 취약: 원격 관리 시 관리자 IP 또는 특정 IP만 접근할 수 있도록 설정되지 않은 경우

**점검방법**:
1. 보안 장비에서 제공하고 있는 메뉴에서 접속 IP나 계정 제한 확인

**조치방법**:
1. 특정 IP나 계정에서만 접속할 수 있도록 설정
2. 원격 관리 시 관리자 및 특정 IP만 접근 허용

**참고사항**:
- 보안 장비에서 제공하는 관리자 접근제한 기능을 통해 관리자 단말기 또는 콘솔 장비의 허용된 IP만을 등록하고 접근을 제한할 수 있음
- 보안 장비 원격 관리를 원칙적으로 금지하나, 부득이 사용해야 하는 경우 원격 접속을 허용할 IP나 계정을 제한하는 등 보안대책을 세워 관리해야 함
- 보안 위협: 보안 장비 원격 관리 시 특정 IP만 접근할 수 있도록 설정하지 않을 경우, 외부에 있는 비인가자가 탈취한 계정을 통해 보안 장비에 접근할 수 있게 되어 보안 장비 설정값을 변경하여 보안 장비를 무력화시킬 위험이 존재함

---

### S-07 보안장비 보안 접속

> 점검ID: S-07 | 위험도: 상 | 카테고리: 접근관리

**점검내용**: 보안 장비 접속 시 암호화 프로토콜을 이용한 접속 여부 점검

**판단기준**:
- ✅ 양호: 보안 장비 접속 시 암호화 통신을 하는 경우
- ❌ 취약: 보안 장비 접속 시 암호화 통신을 하지 않는 경우

**점검방법**:
1. HTTPS 또는 SSH를 통한 접속 확인

**조치방법**:
1. 보안 장비 접속 시, SSL, HTTPS 등의 암호화 접속 활용
2. 제품마다 상이하므로 벤더사에 문의

**참고사항**:
- SSL(Secure Socket Layer): 웹사이트와 브라우저 사이에 전송되는 데이터를 암호화하여 인터넷 연결을 보호하기 위한 표준 기술
- 보안 장비에 대한 원격 접속을 원칙적으로 금지하나, 부득이 원격 접속을 해야 하는 경우 암호화 통신 프로토콜 사용 등 보안대책을 세워 접속해야 함
- 보안 위협: 보안 장비 접속 시 암호화 통신을 하지 않을 경우, 데이터가 평문으로 전송되어 비인가자가 관리자 계정을 탈취할 위험이 존재함

---

### S-08 세션 종료 시간 설정

> 점검ID: S-08 | 위험도: 상 | 카테고리: 접근관리

**점검내용**: 보안 장비의 정책에 Session Timeout 설정을 적용하였는지 점검

**판단기준**:
- ✅ 양호: Session Timeout을 설정한 경우
- ❌ 취약: Session Timeout을 설정하지 않은 경우

**점검방법**:
1. 벤더별 설정값 확인 (대부분의 보안 장비는 기본으로 설정되어 있음)
2. Console, SSL, VPN, SSH 등의 모든 원격 접근에 대한 Session Timeout 설정 확인

**조치방법**:
1. 보안 장비가 제공하는 Session Timeout 기능 활성화 설정
2. Session Timeout 시간을 설정

**참고사항**:
- Session Timeout: 로컬 또는 원격에서 보안 장비에 접속한 사용자가 일정 시간 동안 통신이 없을 시 해당 세션을 종료시키는 설정
- 보안 장비에 접속하여 이행하는 업무 특성을 고려하여 시간을 설정하도록 함
- 보안 위협: Session Timeout 설정을 하지 않을 경우, 보안 장비에 접속한 사용자가 자리 이탈 또는 불완전한 세션 종료로 인해 권한이 없는 사용자가 해당 세션을 재사용하여 인증 없이 보안 장비에 접근함으로써 보안 정책 변경 및 삭제 등 악의적인 행위를 할 위험이 존재함

---

## 3. 패치 관리

### S-09 주기적 보안 패치 및 벤더 권고사항 적용

> 점검ID: S-09 | 위험도: 상 | 카테고리: 패치관리

**점검내용**: 보안 장비 OS 및 보안 기능(IPS, 안티바이러스 등)을 주기적으로 보안 패치 여부 점검

**판단기준**:
- ✅ 양호: 주기적 보안 패치 및 벤더 권고사항이 적용된 경우
- ❌ 취약: 주기적 보안 패치 및 벤더 권고사항이 적용되지 않은 경우

**점검방법**:
1. 보안 장비의 최신 패치 현황 파악 또는 벤더사에 문의하여 최신 패치 현황 파악
2. 주기적으로 업데이트 필요성에 대한 검토 및 조처를 하고 있는지 문서 (정기점검보고서, 검토보고서, 작업계획서 등)를 통해 검토

**조치방법**:
1. 자동 패치 기능 설정 또는 벤더사에 문의하여 수동으로 패치 수행
2. 벤더사에서 주기적으로 제공하는 장비별 최신 취약점 정보를 파악 후 최신 패치 및 업그레이드를 수행

**참고사항**:
- 최신 업데이트 적용 시 시스템 재시작 등의 서비스 영향도 검토 필요
- 보안 위협: 주기적 보안 패치 및 벤더 권고사항을 적용하지 않을 경우, 보안 장비 OS의 취약점을 이용한 공격이나 최신 유해 트래픽에 대한 탐지 및 차단이 제대로 이루어지지 않아 비인가자의 내부 정보시스템 침해 위험이 존재함

## 4. 로그 관리

### S-10 보안장비 로그 설정

> 점검ID: S-10 | 위험도: 중 | 카테고리: 로그관리

**점검내용**: 보안 장비에 로그 설정이 적용되어 있는지 확인하고 로그 정책이 기관 정책에 맞게 적용되어 있는지 점검

**판단기준**:
- ✅ 양호: 기관 정책에 따른 로그 설정된 경우
- ❌ 취약: 기관 정책에 따른 로그 설정이 되지 않은 경우

**점검방법**:
1. 보안 장비의 로그 설정 메뉴 확인

**조치방법**:
1. 기관 정책에 따른 로깅 설정 (각 벤더별 설정 방법이 상이함)
2. 기관 정책에 따른 로깅 설정

**참고사항**:
- 로그 정책 설정: 로그 정책 설정 시 보안 장비에 대한 접근 이력(날짜, 시간, IP, ID, 명령어 이력 등), 보안 장비 성능 이상 유무(CPU, RAM 사용량 등), 보안 장비를 통해 유입되거나 외부로 나가는 트래픽에 대해 로그가 남도록 설정하는 것을 권장
- 세부적인 로깅 설정은 보안 장비 성능에 영향을 미칠 수 있음
- 보안 위협: 로그 설정이 적용되어 있지 않을 경우, 보안 장비에 장애가 발생하거나 침해사고가 발생했을 경우 원인 분석이 어려운 위험이 존재함

### S-12 보안장비 정책 백업 설정

1. 보안 장비에 적용된 정책을 별도의 파일로 보관하는지 확인

2. 보안 장비에 적용된 정책을 별도의 파일로 보관

### S-13 원격 로그 서버 사용

> 점검ID: S-13 | 위험도: 중 | 카테고리: 로그관리

**점검내용**: 보안 장비 원격 로그 설정((r)syslog)이 기관 정책에 맞게 설정되어 있으며 보안 장비 로그를 원격 로그 서버에 별도로 보관하도록 설정되어 있는지 점검

**판단기준**:
- ✅ 양호: 별도의 원격 로그 서버가 구축되어 있고, 원격 로그 서버에 저장될 로그가 기관 정책에 맞게 설정된 경우
- ❌ 취약: 별도의 원격 로그 서버가 구축되어 있지 않거나, 원격 로그 서버에 저장될 로그가 기관 정책에 맞게 설정되지 않은 경우

**점검방법**:
1. 보안 장비 로그 설정 메뉴에서 syslog 설정 확인

**조치방법**:
1. 원격 syslog 로그 수집 서버 설정
2. 로컬에서 별도의 저장매체를 통해 수동으로 로그 백업 (syslog를 지원하지 않는 경우)
3. 기관 정책에 맞게 원격 로그 서버에 저장될 로그 설정 후 보안 장비 로그 설정 메뉴에서 (r)syslog 설정 또는 주기적으로 별도 저장 매체에 백업((r)syslog 미지원일 경우)

**참고사항**:
- 원격 로그 서버: 정보시스템(서버, 네트워크, 보안 장비 등)의 로그를 통합적으로 보관하는 서버
- 보안 위협: 별도의 원격 로그 서버가 구축되어 있지 않거나, 원격 로그 서버에 저장될 로그가 기관 정책에 맞게 설정되어 있지 않을 경우, 비인가자의 공격 및 침입 사고 또는 보안 장비 문제 발생 시 로그가 삭제되거나 변조되어 사고 원인 분석이 어려운 위험이 존재함

### S-14 NTP 및 시각 동기화 설정

> 점검ID: S-14 | 위험도: 중 | 카테고리: 로그관리

**점검내용**: 보안 장비의 NTP 및 시간 동기화 설정 적용 여부 점검

**판단기준**:
- ✅ 양호: NTP 및 시간 동기화 설정이 되어 있는 경우
- ❌ 취약: NTP 및 시간 동기화 설정이 되어 있지 않은 경우

**점검방법**:

**조치방법**:
1. 보안 장비 설정 메뉴에서 NTP 서버 연동 설정
2. 보안 장비 시간 설정에서 NTP 및 시간 동기화 설정 확인

- 보안 위협: NTP 및 시간 동기화 설정이 되어있지 않을 경우, 시스템 간 시간 동기화 미흡으로 보안 사고 및 장애 발생 시 로그에 대한 신뢰도가 떨어질 위험이 존재함

### S-15 정책 관리

- ❌ 취약: 정책에 대한 주기적인 검사를 하지 않고 미사용 및 중복된 정책을 확인 및 제거되지 않은 경우

**점검방법**:
1. 정책에 대한 주기적인 검사로 미사용 & 중복된 정책 확인

   - 보안 장비 정책 입력 시 IP 대신 이름을 사용하도록 함 (예시: 그룹명: OOO부서, OOO팀, OOO팀의 OOO 등)

   - 사용빈도가 높은 정책은 정책 설정 시 상단에 위치하도록 함

**참고사항**:

- 보안 위협: 미사용 및 중복된 정책을 제거하지 않을 경우, 보안 장비 관리자의 업무 편의성 및 효율성이 저하되며 설정되어있는 정책 중 관리자가 인지하지 못한 정책으로 인해 네트워크 보안 체계가 약화 될 위험이 존재함

### S-16 NAT 설정

> 점검ID: S-16 | 위험도: 상 | 카테고리: 기능관리

**점검내용**: 외부 공개 필요성이 없는 정보시스템에 NAT 설정 여부를 점검

**판단기준**:
- ✅ 양호: 외부 공개 필요성이 없는 서버, 단말기 등 정보시스템에 대해 NAT 설정을 적용한 경우
- ❌ 취약: 외부 공개 필요성이 없는 서버, 단말기 등 정보시스템에 대해 NAT 설정을 적용하지 않은 경우

**점검방법**:
1. 공인 IP 확인 사이트 (포털 등)에 접속하여 사용 중인 단말의 IP 확인

**조치방법**:
1. 대부분의 네트워크 보안 제품들은 NAT 기술을 기본으로 채택하고 있으므로 내부 사설 IP 부여 정책에 맞춰 적용하도록 함
2. 외부 공개 필요성이 없는 정보시스템에 대해 공인 IP 지정 여부를 확인하여 사설 IP로 변경한 후 보안 장비에서 NAT 설정을 적용

**참고사항**:
- NAT 사용 목적:
  1) 인터넷의 공인 IP주소 절약
  2) 공공망과 연결되는 사용자들의 고유한 사설망을 침입자들로부터 보호
- 보안 위협: NAT 설정을 하지 않을 경우, 공인 IP를 통해 시스템에 접근이 가능하여 정보 유출, 시스템 파괴, 악성코드 전파 등의 불법적 행위가 발생할 위험이 존재함

### S-17 DMZ 설정

> 점검ID: S-17 | 위험도: 상 | 카테고리: 기능관리

**점검내용**: 내부 네트워크와 외부 서비스 네트워크(DMZ)를 구분하고 있는지 점검

**판단기준**:
- ✅ 양호: DMZ를 구성하여 내부 네트워크를 보호하는 경우
- ❌ 취약: DMZ를 구성하지 않고 사설망에서 외부 공개 서비스를 제공하는 경우

**점검방법**:
1. 네트워크 구성도 또는 방화벽 설정 확인

**조치방법**:
1. 방화벽의 옵션 설정: 각각의 네트워크는 방화벽에 서로 다른 포트를 사용하여 연결하는데 이를 삼각 방화벽 설정(three-legged firewall set-up)이라 함
2. 두 개의 방화벽 사용: DMZ는 두 개의 방화벽 중간에 위치하며, 두 개의 방화벽과 연결됨. 하나의 방화벽은 내부 네트워크와 연결되고 다른 하나는 외부 네트워크와 연결됨. 우연한 설정 실수를 통해 외부 네트워크가 내부 네트워크로 연결할 수 있게 되는 상황을 방지함. 이런 구성 형식을 차단된 서브넷 방화벽(screened-subnet firewall)이라고 함
3. DMZ를 구성하여 내부 네트워크와 외부 서비스 네트워크 분리
4. 물리적(망 분리)으로 내부 네트워크와 외부 서비스 네트워크가 분리되어 있으면 해당 없음

**참고사항**:
- DMZ: 조직의 내부 네트워크와 외부 네트워크 사이에 위치하는 네트워크 망으로 DMZ 내 컴퓨터는 외부 네트워크에만 연결할 수 있도록 하고, 내부 네트워크로는 연결할 수 없도록 구성함
- 보안 위협: DMZ 설정을 통해 내부 네트워크와 외부 서비스 네트워크를 구분하도록 설정되어 있지 않을 경우, 외부 네트워크를 통해서 서비스를 제공받는 악의적인 사용자가 DMZ 내 호스트를 통해 내부 네트워크로 불법적 연결을 시도할 위험이 존재함

---

### S-18 최소한의 서비스만 제공

> 점검ID: S-18 | 위험도: 상 | 카테고리: 기능관리

**점검내용**: 보안 장비에서 필요한 서비스만 제공하고 있는지 점검

**판단기준**:
- ✅ 양호: All Deny 설정 및 보안 장비에 최소 서비스만 허용하는 경우
- ❌ 취약: All Deny 미설정 또는 보안 장비에 불필요한 서비스를 허용하는 경우

**점검방법**:
1. 방화벽에서 허용되지 않은 포트 접속 확인
2. 방화벽 기본 정책인 all deny에 최소 서비스만 허용 확인 (허용된 IP와 서비스 포트만 오픈, 모든 IP 및 서비스 허용 금지)

**조치방법**:
1. 보안 장비에 최소 서비스만 허용하도록 설정함

**참고사항**:
- 방화벽 기본 정책: 방화벽은 기본적으로 All Deny 설정을 적용하며 허용할 IP와 포트만 추가함으로써 관리 포인트를 최소화함
- 허용되지 않은 접속은 모두 차단됨
- 보안 위협: 최소한의 서비스만 제공하지 않을 경우, 비인가자의 침입 또는 악성 소프트웨어 전달 등의 보안 위험이 발생할 위험이 존재함

### S-19 이상징후 탐지 모니터링 수행

> 점검ID: S-19 | 위험도: 상 | 카테고리: 기능관리

**판단기준**:

- ❌ 취약: 이상징후 탐지 모니터링을 수행하고 있지 않은 경우

**점검방법**:
1. 보안 장비의 이상징후 탐지 모니터링 기능 (알람, 이메일, SMS 등)이 설정되어 담당자가 즉시 인지할 방안이 있는지를 점검

**조치방법**:

2. 이상징후 탐지 시 담당자/관리자가 즉시 확인할 수 있도록 모니터링 수행

**참고사항**:
- 보안 사고 미연 방지 및 IT 컴플라이언스 준수
  - 예시 1) 휴일 또는 업무 시간 이외에 비정상적인 패턴으로 중요 문서 또는 고객 정보 DB에 접근한다면 정보 유출 징후가 있다고 판단 가능
  - 예시 2) 퇴사 징후가 의심되는 직원이 휴일이나 업무 시간 외에 비정상적으로 중요 문서나 고객 DB에 접근하는 경우 정보 유출 가능성이 커 이를 사전에 탐지하고 예방할 수 있음
- 보안 위협: 이상징후 탐지 모니터링을 수행하지 않을 경우, 보안 사고를 미리 방지할 수 없는 위험이 존재함

> 점검ID: S-21 | 위험도: 상 | 카테고리: 기능관리

**판단기준**:
- ✅ 양호: 불필요한 SNMP 서비스를 사용하지 않을 경우
- ❌ 취약: 불필요한 SNMP 서비스를 사용할 경우

2. 불필요한 경우 SNMP 서비스 중지

**참고사항**:

- SNMP 서비스를 이용해야 하는 경우 Community String을 유추할 수 없게 설정
- 보안 위협: 불필요한 SNMP 서비스를 사용할 경우, DoS 공격, 보안 장비 성능 저하, 크래쉬, 리로드 등의 공격 위험이 존재함

### S-22 SNMP Community String 복잡성 설정

> 점검ID: S-22 | 위험도: 상 | 카테고리: 기능관리

**점검내용**: SNMP Community String이 유추하기 어렵게 설정되어 있는지 점검

**판단기준**:
- ✅ 양호: SNMP 서비스를 사용하지 않거나, 유추하기 어려운 Community String을 설정한 경우
- ❌ 취약: Community String을 기본값으로 사용하고 있거나, 유추하기 쉬운 Community String을 설정한 경우

**점검방법**:
1. 보안 장비의 SNMP 설정 메뉴에서 Community String 확인

**조치방법**:
1. 보안 장비는 SNMP 취약성이 존재하므로 누구나 추측하기 어렵고 의미가 없는 문자열, 영문자 혼합으로 변경
2. 유추하기 어려운 Community String을 설정

**참고사항**:
- SNMP 버전: v1, v2, v3이 존재. v1, v2는 Community String을 평문 전송하지만, v3는 암호화가 설정되어 해시값으로 전송함
- 보안 위협: SNMP의 Community String을 기본값 또는 유추하기 쉬운 값으로 설정할 경우, 악의적인 사용자가 장비 설정을 쉽게 변경(RW)하여 중요 시스템 정보가 노출될 수 있는 위험이 존재함

### S-23 유해 트래픽 탐지/차단 정책 설정

> 점검ID: S-23 | 위험도: 중 | 카테고리: 기능관리

**점검내용**: 유해 트래픽 탐지/차단 정책 적용 여부 점검

**판단기준**:
- ✅ 양호: 유해 트래픽 탐지/차단 패턴이 적용된 경우
- ❌ 취약: 유해 트래픽 탐지/차단 패턴이 적용되지 않은 경우

**점검방법**:

**Flooding 공격**:
1. TCP/UDP/ICMP Flooding 공격에 대한 탐지 및 차단 패턴 적용 여부를 점검
2. 인터넷 진입구간에 위치한 DDoS 차단시스템에 TCP/UDP/ICMP Flooding 공격에 대한 차단정책 등록 여부를 점검

**Scan 공격**:
1. Port Scan에 대한 탐지 및 차단 패턴 적용 여부를 점검
2. 인터넷 진입 구간에 위치한 DDoS 차단시스템에 Port Scan에 대한 차단 정책 등록 여부를 점검

**해킹툴 공격**:
1. 악성코드에 대한 탐지 및 차단 패턴 적용 여부를 점검

**조치방법**:

**Flooding 공격**:
1. TCP/UDP/ICMP Flooding 공격에 대한 탐지 및 차단 패턴 적용
2. 인터넷 진입구간에 위치한 DDoS 차단시스템에 TCP/UDP/ICMP Flooding 공격에 대한 차단 정책 등록

**Scan 공격**:
1. Port Scan에 대한 탐지 및 차단 패턴 적용
2. 인터넷 진입 구간에 위치한 DDoS 차단시스템에 Port Scan에 대한 차단 정책 등록

**해킹툴 공격**:
1. 주요 해킹 툴에 대한 탐지 및 차단정책 등록

**참고사항**:
- Flooding 공격: 악의적으로 한꺼번에 많은 양의 데이터를 보내 상대방의 서비스를 운용할 수 없게 만드는 서비스 거부 공격
- Scan 공격: 네트워크, 시스템의 활성화된 포트·서비스·취약점을 식별하기 위한 행위
- 해킹툴 공격: 공격자가 상용 해킹툴이나, 자제 제작한 해킹툴을 이용해 시스템을 침투·장악하거나 권한을 획득하는 행위
- 오탐으로 인한 차단 정책 설정 시 서비스 영향 발생
- 보안 위협: 유해 트래픽 탐지/차단 정책이 설정되지 않을 경우, 악성코드가 네트워크를 통해 전파될 우려가 있음
