# 개요 (Overview) 섹션 작성 가이드

백서의 첫인상을 결정하는 가장 중요한 섹션입니다.

## 목적

- 프로젝트의 핵심 가치를 한눈에 전달
- 독자의 관심을 끌어 계속 읽게 유도
- 전체 백서의 방향성 제시

## 포함해야 할 내용

### 1. Executive Summary (요약)
- [ ] 프로젝트 한 줄 소개
- [ ] 해결하려는 문제 (1-2문장)
- [ ] 제안하는 솔루션 (1-2문장)
- [ ] 핵심 차별점 (1-2문장)
- [ ] 목표/비전 (1-2문장)

### 2. 문제 정의
- [ ] 현재 상황의 문제점
- [ ] 문제의 규모/심각성 (데이터 필수)
- [ ] 영향받는 대상

### 3. 시장 기회
- [ ] 시장 규모 (TAM/SAM/SOM)
- [ ] 성장 추세
- [ ] 미충족 수요

## 작성 팁 (투자자 관점)

### DO (해야 할 것)
- **구체적 수치 제시**: "시장 규모 X조원" 대신 "2024년 기준 X조원, 연 Y% 성장"
- **문제의 긴급성 강조**: 왜 지금 해결해야 하는지
- **독자 관점 작성**: "우리는 ~했다" 대신 "사용자는 ~를 얻는다"
- **검증 가능한 출처**: 신뢰할 수 있는 기관의 데이터 인용

### DON'T (피해야 할 것)
- 추상적 표현 ("혁신적인", "최고의")
- 근거 없는 주장
- 기술 용어 남발 (첫 등장 시 설명 필요)
- 장황한 서론

## 예시 구조

```markdown
## 1. 요약

### 핵심 메시지
- **문제**: 국내 물류비용이 GDP의 12%를 차지하며, 라스트마일 비효율로 연간 X조원 손실
- **솔루션**: AI 기반 실시간 라우팅으로 배송 효율 40% 향상
- **차별점**: 특허받은 예측 알고리즘으로 정확도 95% 달성
- **목표**: 2025년까지 국내 물류 시장 점유율 5% 달성

## 2. 문제 정의

### 2.1 현재 상황
2024년 국내 택배 물량은 연간 40억 개를 돌파했습니다(통계청, 2024).
그러나 라스트마일 배송 효율은 10년간 정체되어 있습니다.

- 평균 배송 실패율: 15%
- 재배송 비용: 건당 3,000원
- 연간 재배송 비용: 약 1.8조원
```

## 분량 가이드

| 항목 | 권장 분량 |
|------|-----------|
| Executive Summary | 200-300단어 (1페이지 이내) |
| 문제 정의 | 300-500단어 |
| 시장 기회 | 200-300단어 |
| **전체** | **700-1,100단어** |

## 체크리스트

작성 완료 후 확인:
- [ ] 비전문가도 이해할 수 있는가?
- [ ] 모든 수치에 출처가 있는가?
- [ ] "그래서 뭐?"라는 질문에 답하는가?
- [ ] 다음 섹션을 읽고 싶게 만드는가?

## 안티패턴

### 나쁜 예시
> "우리는 혁신적인 기술로 업계를 선도하는 최고의 솔루션을 제공합니다."

**문제점**: 구체성 없음, 검증 불가, 모든 회사가 할 수 있는 말

### 좋은 예시
> "우리 솔루션은 A사 대비 처리 속도 3배, 비용 40% 절감을 달성했습니다(2024년 PoC 결과)."

**장점**: 구체적 수치, 비교 대상 명확, 검증 가능
