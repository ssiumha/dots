# 해결책 (Solution) 섹션 작성 가이드

문제에 대한 구체적인 해결 방안을 제시하는 섹션입니다.

## 목적

- 제안하는 솔루션의 핵심 개념 전달
- 솔루션이 문제를 어떻게 해결하는지 설명
- 사용자 관점의 가치 제안

## 포함해야 할 내용

### 1. 솔루션 개요
- [ ] 솔루션의 핵심 개념 (1-2문단)
- [ ] 작동 원리 요약
- [ ] 기존 방식과의 차이점

### 2. 핵심 기능
- [ ] 3-5개 핵심 기능 나열
- [ ] 각 기능의 사용자 이점
- [ ] 기능 간 연관성

### 3. 사용 사례
- [ ] 2-3개 구체적 시나리오
- [ ] 입력 → 처리 → 결과 흐름
- [ ] Before/After 비교

### 4. 사용자 가치
- [ ] 정량적 이점 (시간 절약, 비용 절감)
- [ ] 정성적 이점 (편의성, 안정성)

## 작성 팁 (투자자 관점)

### DO (해야 할 것)
- **사용자 여정 중심**: 기능 나열보다 사용자 경험 설명
- **Before/After 명확화**: 기존 방식 vs 우리 솔루션 비교
- **정량적 가치**: "효율적"보다 "40% 시간 절감"
- **실제 사례**: 가상이라도 구체적인 시나리오

### DON'T (피해야 할 것)
- 기능 목록만 나열
- 기술 용어 과다 사용
- 제품 홍보 톤
- 경쟁사 비방

## 예시 구조

```markdown
## 3. 해결책

### 3.1 솔루션 개요
[프로젝트명]은 AI 기반 실시간 라우팅 플랫폼입니다.
기존 배송 시스템은 정적인 경로를 사용하지만,
우리 솔루션은 실시간 교통, 날씨, 수요를 분석하여
최적 경로를 동적으로 생성합니다.

### 3.2 핵심 기능

**1. 실시간 경로 최적화**
- 30초마다 최적 경로 재계산
- 교통 상황 반영으로 지연 최소화
- 사용자 이점: 평균 배송 시간 25% 단축

**2. 수요 예측**
- 과거 데이터 기반 배송량 예측
- 사전 자원 배치 최적화
- 사용자 이점: 피크 시간 대응력 향상

**3. 자동 스케줄링**
- 드라이버 배정 자동화
- 업무 균형 최적화
- 사용자 이점: 관리 인력 50% 절감

### 3.3 사용 사례

**시나리오: 출퇴근 시간 대량 배송**

| 단계 | 기존 방식 | 우리 솔루션 |
|------|-----------|-------------|
| 경로 설정 | 고정 경로 | 실시간 최적화 |
| 교통 대응 | 수동 우회 | 자동 재라우팅 |
| 결과 | 2시간 지연 | 정시 도착 |

### 3.4 사용자 가치

**정량적 이점**
- 배송 시간: 평균 25% 단축
- 운영 비용: 30% 절감
- 배송 실패율: 15% → 5%

**정성적 이점**
- 실시간 모니터링으로 안심
- 예측 가능한 배송 시간
- 관리 부담 감소
```

## 분량 가이드

| 항목 | 권장 분량 |
|------|-----------|
| 솔루션 개요 | 200-300단어 |
| 핵심 기능 | 300-500단어 |
| 사용 사례 | 200-300단어 |
| 사용자 가치 | 150-200단어 |
| **전체** | **850-1,300단어** |

## 체크리스트

작성 완료 후 확인:
- [ ] 문제 정의 섹션의 문제를 해결하는가?
- [ ] 비전문가도 작동 원리를 이해하는가?
- [ ] 사용자 이점이 명확한가?
- [ ] 구체적인 수치/사례가 있는가?

## 안티패턴

### 나쁜 예시
> "우리 솔루션은 다음 기능을 제공합니다:
> - 기능 A
> - 기능 B
> - 기능 C"

**문제점**: 기능 나열만 있고, 왜 필요한지/어떤 가치인지 없음

### 좋은 예시
> "기능 A를 통해 사용자는 기존 2시간 걸리던 작업을 10분에 완료할 수 있습니다.
> 이는 B사 대비 3배 빠른 속도입니다."

**장점**: 기능 + 사용자 가치 + 비교 기준 포함
