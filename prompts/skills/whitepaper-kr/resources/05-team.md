# 팀 (Team) 섹션 작성 가이드

실행력과 신뢰성을 보여주는 중요한 섹션입니다.

## 목적

- 팀의 역량과 경험 제시
- 이 팀이 성공할 수 있는 이유 설명
- 신뢰성 구축

## 포함해야 할 내용

### 1. 핵심 팀원
- [ ] 이름, 직책
- [ ] 관련 경력 (2-3줄)
- [ ] 핵심 전문성
- [ ] LinkedIn/GitHub (선택)

### 2. 팀 구성
- [ ] 현재 팀 규모
- [ ] 조직 구조
- [ ] 채용 계획

### 3. 자문단/어드바이저
- [ ] 이름, 소속
- [ ] 자문 분야
- [ ] 관련 경력

### 4. 파트너십
- [ ] 전략적 파트너
- [ ] 투자자
- [ ] 기술 파트너

## 작성 팁 (투자자 관점)

### DO (해야 할 것)
- **관련 경력 강조**: 이 사업과 직접 연관된 경험
- **성과 중심**: "~했다"보다 "~를 달성했다"
- **팀 상호보완성**: 다양한 전문성 균형
- **검증 가능**: LinkedIn 등 확인 가능한 정보

### DON'T (피해야 할 것)
- 과장된 직책/경력
- 관련 없는 경력 나열
- 팀원 사진 없는 긴 설명
- 자문단 과도하게 부풀리기

## 예시 구조

```markdown
## 6. 팀

### 6.1 핵심 팀원

**김철수** - CEO & Co-founder
- 경력: 네이버 PM 5년, 카카오 사업개발 3년
- 성과: 월간 사용자 100만 서비스 런칭 경험
- 전문성: 프로덕트 전략, 사업 개발
- LinkedIn: linkedin.com/in/xxx

**이영희** - CTO & Co-founder
- 경력: 구글 소프트웨어 엔지니어 7년, 스타트업 CTO 3년
- 성과: 대규모 분산 시스템 설계 (DAU 1,000만)
- 전문성: 백엔드 아키텍처, 클라우드 인프라
- GitHub: github.com/xxx

**박지민** - Head of AI
- 경력: KAIST AI 박사, 삼성리서치 연구원 4년
- 성과: 논문 15편 (CVPR/NeurIPS), 특허 3건
- 전문성: 컴퓨터 비전, 딥러닝 최적화
- Google Scholar: scholar.google.com/xxx

### 6.2 팀 구성

**현재 조직** (총 12명)
| 부서 | 인원 | 역할 |
|------|------|------|
| 경영 | 2명 | CEO, COO |
| 개발 | 6명 | CTO, 백엔드 3, 프론트엔드 2 |
| AI | 3명 | Head of AI, ML Engineer 2 |
| 디자인 | 1명 | Product Designer |

**채용 계획** (2025년)
- Q1: ML Engineer 2명, 백엔드 2명
- Q2: 세일즈 2명, 마케팅 1명
- 연말 목표: 20명

### 6.3 자문단

**기술 자문**
- **김교수** (서울대 컴퓨터공학)
  - AI/ML 분야 전문가, 논문 100+편
  - 자문 분야: AI 연구 방향, 기술 검증

**사업 자문**
- **이대표** (전 A사 대표)
  - B2B SaaS 스케일업 경험
  - 자문 분야: 사업 전략, 투자 유치

**업계 자문**
- **박이사** (X그룹 디지털 총괄)
  - 대기업 디지털 전환 20년 경력
  - 자문 분야: 엔터프라이즈 세일즈, 파트너십

### 6.4 파트너십

**전략적 파트너**
- **A사**: 기술 협력, API 연동
- **B사**: 공동 마케팅, 채널 파트너

**투자자**
- Seed: XX벤처스, YY파트너스 (X억원)

**기술 파트너**
- AWS: Activate Program 참여
- NVIDIA: Inception Program 멤버
```

## 분량 가이드

| 항목 | 권장 분량 |
|------|-----------|
| 핵심 팀원 | 300-500단어 |
| 팀 구성 | 100-200단어 |
| 자문단 | 150-250단어 |
| 파트너십 | 100-150단어 |
| **전체** | **650-1,100단어** |

## 팀원 소개 템플릿

```markdown
**[이름]** - [직책]
- 경력: [주요 경력 1-2개]
- 성과: [정량적 성과 1개]
- 전문성: [핵심 역량 2-3개]
- [링크]: [URL]
```

## 체크리스트

작성 완료 후 확인:
- [ ] 핵심 직책이 모두 채워져 있는가?
- [ ] 관련 경력이 명확한가?
- [ ] 팀의 상호보완성이 보이는가?
- [ ] 경력이 검증 가능한가?

## 프로젝트 유형별 강조점

### AI 프로젝트
- AI/ML 전문가 경력 상세
- 논문, 특허 실적
- 연구기관 연계

### 블록체인 프로젝트
- 블록체인 개발 경험
- 오픈소스 기여
- 보안/암호학 전문성

### B2B SaaS
- 도메인 전문성
- 세일즈/마케팅 역량
- 엔터프라이즈 경험

## 안티패턴

### 나쁜 예시
> "홍길동 - CEO
> 서울대 졸업, 여러 회사 경험, 열정적인 리더"

**문제점**: 구체성 없음, 관련성 불명확, 성과 없음

### 좋은 예시
> "홍길동 - CEO & Co-founder
> - 경력: 쿠팡 물류 PM 4년 (물류 효율 30% 개선 프로젝트 리드)
> - 성과: 스타트업 A사 공동창업, 3년 만에 매각 (XX억원)
> - 전문성: 물류 도메인, 스케일업 경험"

**장점**: 관련 경력, 정량적 성과, 검증 가능
