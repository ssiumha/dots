# 비즈니스 모델 (Business Model) 섹션 작성 가이드

> **적용 대상**: 일반 기술 프로젝트, AI 프로젝트
> **블록체인 프로젝트는 토크노믹스 섹션 사용**

수익 구조와 시장 전략을 설명하는 섹션입니다.

## 목적

- 수익 창출 방식 명확화
- 가격 전략 제시
- 시장 기회 정량화

## 포함해야 할 내용

### 1. 수익 구조
- [ ] 주요 수익원
- [ ] 수익 모델 유형 (구독/거래/광고 등)
- [ ] 수익원별 비중

### 2. 가격 정책
- [ ] 플랜별 가격
- [ ] 포함 기능
- [ ] 가격 책정 근거

### 3. 시장 규모
- [ ] TAM (Total Addressable Market)
- [ ] SAM (Serviceable Addressable Market)
- [ ] SOM (Serviceable Obtainable Market)

### 4. 성장 전략
- [ ] 고객 획득 전략
- [ ] 리텐션 전략
- [ ] 확장 전략

## 작성 팁 (투자자 관점)

### DO (해야 할 것)
- **유닛 이코노믹스 제시**: CAC, LTV, 회수 기간
- **가격 근거 설명**: 왜 이 가격인지
- **시장 규모 출처 명시**: 신뢰할 수 있는 리서치
- **단계별 전략**: 현재 → 1년 → 3년

### DON'T (피해야 할 것)
- 비현실적 시장 점유율 가정
- 경쟁 무시한 가격 책정
- 근거 없는 성장률
- 복잡한 수익 구조

## 예시 구조

```markdown
## 8. 비즈니스 모델

### 8.1 수익 구조

**주요 수익원**

| 수익원 | 모델 | 예상 비중 |
|--------|------|-----------|
| SaaS 구독 | 월/연 구독 | 70% |
| API 과금 | 사용량 기반 | 20% |
| 컨설팅 | 프로젝트 기반 | 10% |

**SaaS 구독 (핵심 수익원)**
- 월간/연간 구독 모델
- 사용자 수 또는 기능 기반 과금
- 연간 구독 시 20% 할인

**API 과금**
- 호출 횟수 기반 과금
- 월간 무료 한도 제공
- 초과 사용량 당 과금

### 8.2 가격 정책

| 플랜 | 월 가격 | 사용자 | 주요 기능 |
|------|---------|--------|-----------|
| Starter | 무료 | 3명 | 기본 기능 |
| Pro | ₩99,000 | 10명 | 고급 기능, API |
| Business | ₩299,000 | 50명 | 전체 기능, 우선 지원 |
| Enterprise | 별도 문의 | 무제한 | 맞춤형, 전담 매니저 |

**가격 책정 근거**
- 경쟁사 대비 30% 저렴
- 기능당 가치 기준 산정
- 고객 인터뷰 기반 수용 가격

### 8.3 시장 규모

```
┌─────────────────────────────────────────────┐
│                    TAM                       │
│              전체 시장: 10조원               │
│    ┌─────────────────────────────────┐      │
│    │              SAM                 │      │
│    │        서비스 가능: 2조원        │      │
│    │    ┌───────────────────────┐    │      │
│    │    │         SOM           │    │      │
│    │    │   목표 시장: 200억원   │    │      │
│    │    └───────────────────────┘    │      │
│    └─────────────────────────────────┘      │
└─────────────────────────────────────────────┘
```

| 구분 | 규모 | 산출 근거 |
|------|------|-----------|
| TAM | 10조원 | 국내 전체 XX 소프트웨어 시장 (Gartner, 2024) |
| SAM | 2조원 | 중소기업 대상 SaaS 시장 (IDC, 2024) |
| SOM | 200억원 | 초기 타겟 1,000사 × ARPU 200만원 |

**시장 성장률**
- 연평균 성장률 (CAGR): 15%
- 2027년 예상 TAM: 15조원

### 8.4 유닛 이코노믹스

| 지표 | 목표 | 현재 |
|------|------|------|
| CAC (고객 획득 비용) | 50만원 | 80만원 |
| ARPU (객단가) | 200만원/년 | 150만원/년 |
| LTV (고객 생애 가치) | 600만원 | 400만원 |
| LTV/CAC | 12x | 5x |
| Payback Period | 3개월 | 6개월 |
| Churn Rate | 5%/월 | 8%/월 |

### 8.5 성장 전략

**Phase 1: 초기 고객 확보 (0-1년)**
- 직접 영업 (엔터프라이즈)
- 콘텐츠 마케팅 (SEO, 블로그)
- 무료 체험 → 유료 전환

**Phase 2: 스케일업 (1-3년)**
- 파트너 채널 확대
- 마케팅 자동화
- 제품 주도 성장 (PLG)

**Phase 3: 확장 (3년+)**
- 해외 시장 진출
- 인접 시장 확장
- M&A 검토
```

## 분량 가이드

| 항목 | 권장 분량 |
|------|-----------|
| 수익 구조 | 200-300단어 |
| 가격 정책 | 150-250단어 |
| 시장 규모 | 200-300단어 |
| 유닛 이코노믹스 | 100-150단어 |
| 성장 전략 | 150-200단어 |
| **전체** | **800-1,200단어** |

## 체크리스트

작성 완료 후 확인:
- [ ] 수익 모델이 명확한가?
- [ ] 가격이 경쟁력 있는가?
- [ ] 시장 규모에 출처가 있는가?
- [ ] 유닛 이코노믹스가 건전한가?

## 수익 모델 유형

| 유형 | 설명 | 적합한 프로젝트 |
|------|------|-----------------|
| SaaS 구독 | 월/연 정기 구독 | B2B 소프트웨어 |
| 사용량 기반 | API 호출, 처리량 | 인프라, AI |
| 거래 수수료 | 거래당 수수료 | 마켓플레이스 |
| 프리미엄 | 기본 무료, 고급 유료 | 소비자 앱 |
| 라이선스 | 일회성 구매 | 온프레미스 |

## 안티패턴

### 나쁜 예시
> "시장 규모가 크고 성장하고 있어서 우리도 성장할 것입니다."

**문제점**: 논리 비약, 구체성 없음

### 좋은 예시
> "TAM 10조원 중 우리 타겟인 중소기업 SaaS 시장(SAM)은 2조원입니다.
> 첫 해 목표인 1,000사 확보 시 SOM 200억원을 달성하며,
> 이는 SAM의 1%로 보수적인 목표입니다."

**장점**: 단계적 논리, 출처 기반, 현실적 목표
