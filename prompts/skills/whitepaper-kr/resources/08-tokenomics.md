# 토크노믹스 (Tokenomics) 섹션 작성 가이드

> **적용 대상**: 블록체인 프로젝트 전용
> **일반/AI 프로젝트는 비즈니스 모델 섹션 사용**

토큰 경제 모델을 설명하는 블록체인 프로젝트의 핵심 섹션입니다.

## 목적

- 토큰의 유틸리티와 가치 제안 설명
- 공정한 분배 구조 제시
- 장기적 토큰 경제 지속 가능성 입증

## 포함해야 할 내용

### 1. 토큰 개요
- [ ] 토큰명, 티커
- [ ] 토큰 표준 (ERC-20, SPL 등)
- [ ] 총 발행량

### 2. 토큰 유틸리티
- [ ] 거버넌스 기능
- [ ] 스테이킹/보상
- [ ] 수수료/결제
- [ ] 기타 유틸리티

### 3. 토큰 분배
- [ ] 항목별 비율
- [ ] 베스팅 스케줄
- [ ] 언락 일정

### 4. 토큰 경제 모델
- [ ] 인플레이션/디플레이션
- [ ] 소각 메커니즘
- [ ] 보상 구조

## 작성 팁 (투자자 관점)

### DO (해야 할 것)
- **명확한 유틸리티**: 왜 토큰이 필요한지
- **공정한 분배**: 팀/투자자 비율 적정
- **긴 베스팅**: 장기 인센티브 정렬
- **투명한 언락**: 예측 가능한 공급

### DON'T (피해야 할 것)
- 유틸리티 없는 토큰
- 팀/투자자 과다 배분 (>40%)
- 짧은 베스팅 (클리프 없음)
- 복잡한 토큰 구조

## 예시 구조

```markdown
## 6. 토크노믹스

### 6.1 토큰 개요

| 항목 | 내용 |
|------|------|
| 토큰명 | ProjectToken |
| 티커 | $PTK |
| 표준 | ERC-20 (Ethereum) |
| 총 발행량 | 1,000,000,000 (10억 개) |
| 초기 유통량 | 150,000,000 (1.5억 개, 15%) |

### 6.2 토큰 유틸리티

**1. 거버넌스 (Governance)**
- 프로토콜 업그레이드 투표
- 파라미터 변경 제안
- 커뮤니티 자금 배분 결정
- 1 PTK = 1 투표권

**2. 스테이킹 (Staking)**
- 네트워크 보안 참여
- 스테이킹 보상 획득 (APY 8-15%)
- 거버넌스 참여 자격

**3. 수수료 (Fees)**
- 플랫폼 이용 수수료 지불
- 스테이킹 시 수수료 할인 (최대 50%)

**4. 보상 (Rewards)**
- 유동성 공급 보상
- 기여자 보상
- 에어드랍

### 6.3 토큰 분배

```
            토큰 분배 (총 10억 개)

     ┌─────────────────────────────────┐
     │       생태계 (30%)              │
     │        300,000,000              │
     ├─────────────────────────────────┤
     │   투자자 (20%)  │   팀 (15%)   │
     │   200,000,000   │  150,000,000  │
     ├─────────────────┼───────────────┤
     │  커뮤니티(20%)  │   재단(15%)   │
     │  200,000,000    │  150,000,000  │
     └─────────────────┴───────────────┘
```

| 항목 | 비율 | 수량 | 용도 |
|------|------|------|------|
| 생태계 | 30% | 300M | 스테이킹 보상, 유동성 인센티브 |
| 투자자 | 20% | 200M | Seed, Private, Public |
| 커뮤니티 | 20% | 200M | 에어드랍, 그랜트, 버그 바운티 |
| 팀 | 15% | 150M | 핵심 팀, 어드바이저 |
| 재단 | 15% | 150M | 운영, 개발, 마케팅 |

### 6.4 베스팅 스케줄

| 항목 | 클리프 | 베스팅 | TGE 언락 |
|------|--------|--------|----------|
| 팀 | 12개월 | 36개월 | 0% |
| 투자자 (Seed) | 6개월 | 24개월 | 0% |
| 투자자 (Private) | 3개월 | 18개월 | 5% |
| 커뮤니티 | 없음 | 48개월 | 10% |
| 생태계 | 없음 | 60개월 | 5% |
| 재단 | 6개월 | 48개월 | 0% |

**언락 타임라인**

```

        │   │   │   │   │   │   │   │   │   │   │

```

### 6.5 토큰 경제 모델

**인플레이션**
- 연간 인플레이션: 5% (스테이킹 보상)
- 점진적 감소: 연간 0.5% 감소
- 최종 인플레이션: 2%

**디플레이션 메커니즘 (소각)**
- 수수료 소각: 거래 수수료의 50% 소각
- 바이백 소각: 분기별 수익의 20% 소각
- 목표: 순 인플레이션 0% 달성

**스테이킹 경제**
| 스테이킹 비율 | APY |
|---------------|-----|
| 0-20% | 15% |
| 20-40% | 12% |
| 40-60% | 10% |
| 60%+ | 8% |

### 6.6 토큰 세일

| 라운드 | 가격 | 모금액 | FDV |
|--------|------|--------|-----|
| Seed | $0.01 | $2M | $10M |
| Private | $0.02 | $5M | $20M |
| Public | $0.03 | $3M | $30M |
| **총계** | - | **$10M** | - |
```

| 베스팅 스케줄 | 150-200단어 |

## 체크리스트

작성 완료 후 확인:
- [ ] 토큰 유틸리티가 명확한가?
- [ ] 분배가 공정한가? (팀+투자자 < 40%)
- [ ] 베스팅이 충분히 긴가? (팀 3년+)
- [ ] 소각/인플레이션 균형이 있는가?

## 업계 표준 비교

| 항목 | 권장 범위 | 주의 필요 |
|------|-----------|-----------|
| 팀 + 어드바이저 | 15-20% | >25% |
| 투자자 (전체) | 15-25% | >30% |
| 커뮤니티/생태계 | 40-60% | <30% |
| 팀 베스팅 | 3-4년 | <2년 |
| 팀 클리프 | 1년 | <6개월 |
| TGE 언락 (팀) | 0% | >10% |

## 안티패턴

### 나쁜 예시
> "토큰 분배: 팀 30%, 투자자 35%, 기타 35%"

**문제점**: 팀+투자자 65%로 과다, 커뮤니티 비율 낮음

### 좋은 예시
> "토큰 분배: 생태계 30%, 커뮤니티 20%, 투자자 20%, 팀 15%, 재단 15%
> 팀 토큰은 1년 클리프 후 3년간 선형 베스팅"

**장점**: 균형 잡힌 분배, 긴 베스팅, 투명한 스케줄
